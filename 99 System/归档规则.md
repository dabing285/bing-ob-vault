# 归档规则

## 默认协作原则
- 用户负责输入，不负责分类。
- 助手负责判断场景、分类、提炼、沉淀、复用。
- 用户可以口述、碎片表达、想到哪说到哪，不需要先整理格式。
- 助手需要把输入转化为可长期使用的知识资产，而不只是原样记录。

## 助手默认要做的 5 件事
1. 判断这段内容属于什么场景。
2. 判断它值不值得长期保留。
3. 判断里面有没有可复用经验、规律、方法。
4. 判断是否包含待办或下一步动作。
5. 决定最终落到哪个文件或多个文件。

## 场景分类
### 1. 今日日记
适用：当天发生的事情、状态、想法、进展。
默认落点：`01 Daily/当天日期.md`

### 2. 情绪 / 状态
适用：疲惫、焦虑、失眠、兴奋、烦躁、能量高低。
默认先落：`01 Daily/当天日期.md`
如形成稳定模式，再补充到：`04 Areas/精力与状态.md`

### 3. 灵感 / 碎片想法
适用：临时想法、突然冒出的判断、未成熟的点子。
默认先落：`00 Inbox/Captures.md`
如当天就清晰，可直接进 `01 Daily/当天日期.md`
如具备长期价值，再沉淀到 `04 Areas/` 或 `05 Content/`

### 4. 工作复盘
适用：客户沟通、成交、卡点、工作进展、项目判断。
默认落：`01 Daily/当天日期.md`
如可复用，再补充到对应 `04 Areas/` 主题页。

### 5. 客户洞察
适用：某类客户在意什么、为什么成交、常见问题、常见需求。
默认先落：`01 Daily/当天日期.md`
提炼后沉淀到：
- `04 Areas/商业计划书代写.md`
- `04 Areas/客户沟通.md`
- `04 Areas/获客渠道.md`

### 6. 方法论 / 规律 / 经验
适用：反复出现的模式、可复用打法、稳定判断。
默认落：`04 Areas/对应主题.md`
必要时在 Daily 留来源记录。

### 7. 内容素材 / 个人IP素材
适用：选题、观点、金句、表达、案例素材、脚本想法。
默认落：`05 Content/对应文件.md`
常用文件包括：
- `05 Content/选题库.md`
- `05 Content/金句库.md`
- `05 Content/案例素材.md`
- `05 Content/文章草稿.md`
- `05 Content/视频脚本.md`

### 8. 周复盘 / 月复盘
适用：阶段复盘、周期总结、趋势判断。
默认落：
- 周复盘：`02 Weekly/当周.md`
- 月复盘：`03 Monthly/当月.md`

### 9. 待办事项
适用：明确行动项、下一步、需要跟进的事项。
默认处理：从原始输入中抽取，优先写入对应 Daily / Weekly / Monthly 的“下一步”部分。

### 10. 临时噪音
适用：没有长期价值、没有复用价值、只是瞬时表达的内容。
默认处理：可只保留在 Daily 或 Inbox，不强行沉淀到 Areas。

## 默认判断顺序
助手收到输入后，默认按下面顺序处理：
1. 先判断是不是“当天发生的事” → 是则先进 Daily。
2. 再判断有没有“长期规律 / 方法 / 洞察” → 有则补进 Areas。
3. 再判断有没有“可对外输出的表达价值” → 有则补进 Content。
4. 再判断是否有待办 → 有则抽出下一步。

## 默认归档原则
- 先进 Daily，能沉淀的再进 Areas，能发的再进 Content。
- 不要求用户先分类，分类责任在助手。
- 不把所有内容都硬沉淀，避免知识库被噪音污染。
- 不把 Obsidian 当 CRM。
- 客户全量跟进、项目进度、经营台账，优先放飞书；高价值客户洞察和案例提炼，沉淀到 Obsidian。
- 只要是正式归档，默认写入 Obsidian 同步仓库对应目录，不写到助手运行工作区代替正式归档。
- 若因临时处理先写到其他位置，助手必须补写回 Obsidian 正式目录，并明确说明。

## 用户常用口令与默认动作
### “记一下”
默认：写入 Daily。

### “沉淀一下 / 提炼一下 / 总结一下”
默认：写入 Areas，并在必要时保留 Daily 来源。

### “记到内容池 / 这是个选题 / 这个以后能发”
默认：写入 Content。

### “帮我复盘”
默认：根据时间范围写入 Daily / Weekly / Monthly。

### 用户没有明确说类型
默认：由助手自主判断，不再反问用户做分类。

## 长期执行要求
- 助手需要主动替用户减轻认知负担。
- 助手不能把“归档判断”再甩回给用户。
- 助手要持续识别可复用经验、稳定偏好、长期规律，并逐步把碎片输入转化为系统资产。
